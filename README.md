## 安裝擴充功能

> 此擴充功能為手動安裝（開發者模式），不需透過 Chrome Web Store。

### 步驟一：下載原始碼

將 `tickethelper` 資料夾下載至本機任意位置，**請勿移動或刪除資料夾**，擴充功能載入後會參照此路徑。

### 步驟二：開啟 Chrome 擴充功能管理頁

在網址列輸入：

```
chrome://extensions
```

### 步驟三：啟用開發者模式

點擊右上角的 **「開發人員模式」** 切換開關，將其開啟（變為藍色）。

![開發者模式示意](https://i.imgur.com/placeholder.png)

### 步驟四：載入擴充功能

1. 點擊左上角出現的 **「載入未封裝項目」** 按鈕
2. 瀏覽並選取 `tickethelper` 資料夾（內含 `manifest.json` 的那層）
3. 點擊「選取資料夾」

### 步驟五：確認安裝成功

擴充功能清單中出現 **「搶票助手 Pro」** 即代表安裝成功。

建議將擴充功能圖示固定到工具列：點擊瀏覽器右上角的拼圖 🧩 圖示 → 點擊搶票助手 Pro 旁的釘選圖示。

### 日後更新版本

收到新版檔案後：
1. 將新的 `tickethelper` 資料夾覆蓋舊檔案
2. 回到 `chrome://extensions`
3. 點擊搶票助手 Pro 卡片上的 **「重新整理」🔄 圖示**

## 介面說明

點擊瀏覽器工具列上的 🎫 圖示，會以**獨立視窗**開啟控制面板（不會因點擊外側而自動關閉）。

### 全域開關

面板頂部的 **「啟用腳本注入」** 開關，可一鍵暫停 / 恢復所有平台的腳本執行，無需卸載擴充功能。

### 平台切換

- 🔴 **KKTIX** — 切換至 KKTIX 控制面板
- 🟢 **Tixcraft** — 切換至 Tixcraft 控制面板

### 子 Tab 說明

每個平台均有三個子頁籤：

| 頁籤 | 說明 |
|------|------|
| 📝 基礎設定 | 購買數量、票種 / 區域選擇等主要設定 |
| ⚙️ 進階設定 | 執行邏輯、重整延遲等行為設定 |
| ▶️ 執行控制 | 開始 / 停止搶票、查看即時執行紀錄 |

## KKTIX 使用教學

### 第一步：填寫基礎設定

前往 **📝 基礎設定** 頁籤：

#### 1. 購買數量

填入想購買的票數（1–10）。

#### 2. 選擇票種

**方式 A（推薦）：抓取票種**

1. 先在瀏覽器開啟你要搶購的 KKTIX 活動票券選擇頁面
2. 點擊 **「🔄 抓取票種」** 按鈕
3. 頁面上的票種會自動列出，勾選想要的票種

> **優先順序**：數字徽章代表優先序，**先勾選的會先被選取**。  
> 若第一順位無法選購，會自動往下找第二順位，以此類推。

**方式 B（備用）：手動輸入**

展開 **「✏️ 手動輸入（備用）」**，在文字框內輸入票種關鍵字，多個以半形逗號區隔：

```
2,880, 4,480
```

#### 3. 儲存設定

點擊 **「💾 儲存設定」** 按鈕。

---

### 第二步：設定進階選項（選填）

展開 **「⚙️ 進階設定（選填）」**：

- **會員代碼**：若票種需要輸入會員代碼，填入此處，流程中會自動填入
- **問題答案**：若購票頁有文字驗證問題（且沒有 Google reCaptcha），填入答案

---

### 第三步：設定執行邏輯（選填）

前往 **⚙️ 進階設定** 頁籤：

| 設定項目 | 說明 |
|----------|------|
| 找不到符合票種時 | `重新整理頁面`（預設）：找不到票種就重整繼續搶；`選取第一個可購票種`：不管符不符合直接買 |
| 重整延遲（秒） | 每次重整前等待的秒數，預設 1 秒，可輸入 0.1 秒 |

點擊 **「💾 儲存設定」**。

---

### 第四步：開始搶票

1. 在瀏覽器開啟目標活動的 KKTIX **票券選擇頁面**
2. 切換到 **▶️ 執行控制** 頁籤
3. 點擊 **「▶ 開始搶票」**
4. 狀態列會顯示目前進度，執行紀錄會即時更新

> 腳本會自動執行以下步驟：
> 1. 選擇指定票種（依優先順序）
> 2. 點擊「+」設定購買數量
> 3. 勾選同意條款
> 4. 點擊「立即購買」
> 5. 到達結帳頁後，系統會發送 Discord 通知

### 停止搶票

點擊 **「⏹ 停止」** 即可隨時中斷流程。

## Tixcraft 使用教學

### 第一步：填寫基礎設定

前往 **📝 基礎設定** 頁籤：

#### 1. 購買數量

填入想購買的票數（1–10）。

#### 2. 場次關鍵字

輸入想選擇的場次關鍵字，腳本會自動找出符合的場次：

```
2026/03/15
```

> 只要輸入場次日期或名稱的**一部分**即可，例如 `03/15` 也能匹配 `2026/03/15 台北場`。

#### 3. 區域關鍵字

輸入想搶購的區域名稱，多個以半形逗號分隔，**依序優先選取**：

```
A區, 搖滾區, VIP
```

> 第一個會最優先嘗試，若已售完或找不到，才會嘗試下一個。

#### 4. 儲存設定

點擊 **「💾 儲存設定」**。

---

### 第二步：設定進階選項（選填）

展開 **「⚙️ 進階設定（選填）」**：

| 設定項目 | 說明 |
|----------|------|
| 目標網址 | 若擴充功能無法辨識頁面類型，會自動跳轉至此活動網址重試 |
| 區域排除關鍵字 | 含有這些詞的區域會被跳過，例如：`輪椅, 身障, Restricted View` |
| 問題驗證答案 | 部分活動有答題驗證（`/ticket/verify/` 頁面），填入答案可自動通過 |
| OCR API 網址 | 驗證碼辨識服務的位址（詳見 [OCR Server 設定](#ocr-server-設定tixcraft-驗證碼辨識)） |

---

### 第三步：設定執行邏輯（選填）

前往 **⚙️ 進階設定** 頁籤：

| 設定項目 | 說明 |
|----------|------|
| 找不到符合場次時 | `重新整理頁面`（預設）/ `選取第一個可購場次` |
| 找不到符合區域時 | `重新整理頁面`（預設）/ `選取第一個可購區域` |
| 重整延遲（秒） | 每次重整前等待的秒數，預設 1 秒 |

點擊 **「💾 儲存設定」**。

---

### 第四步：啟動 OCR Server（必要）

Tixcraft 結帳頁含有**圖形驗證碼**，需要本機 OCR 伺服器辨識。請先依照 [OCR Server 設定](#ocr-server-設定tixcraft-驗證碼辨識) 啟動服務。

---

### 第五步：開始搶票

1. 在瀏覽器開啟 Tixcraft 活動頁面（活動詳情頁 或 場次選擇頁均可）
2. 切換到 **▶️ 執行控制** 頁籤
3. 確認 **OCR Server** 狀態顯示為 🟢（若未顯示，點擊 🔄 重新檢查）
4. 點擊 **「▶ 開始搶票」**

> 腳本會自動執行以下步驟：
> 1. **活動詳情頁**（`/activity/detail`）：等待場次列表，依關鍵字選擇場次
> 2. **場次選擇頁**（`/activity/game`）：等待場次列表，依關鍵字選擇場次
> 3. **區域選擇頁**（`/ticket/area`）：依優先順序選擇區域，跳過排除關鍵字及剩餘數量不足的區域
> 4. **驗證問題頁**（`/ticket/verify`）：自動填入設定的答案
> 5. **驗證碼結帳頁**（`/ticket/ticket` 或 `/ticket/checkout`）：呼叫 OCR 辨識驗證碼並自動填入送出

若任何步驟失敗，會依設定自動重整並重試。

### 停止搶票

點擊 **「⏹ 停止」** 即可隨時中斷流程。

## OCR Server 設定（Tixcraft 驗證碼辨識）

> 此服務僅 **Tixcraft** 需要。KKTIX 不需要啟動 OCR Server。

OCR Server 是一個本機服務，負責辨識 Tixcraft 的圖形驗證碼。

### 步驟一：安裝 Python

#### 1. 下載 Python

前往 Python 官方網站下載最新版本（需要 Python 3.12 或以上版本）：

👉 **[https://www.python.org/downloads/](https://www.python.org/downloads/)**

點擊 **「Download Python 3.x.x」** 按鈕下載安裝程式。

#### 2. 安裝 Python

執行下載的安裝程式，**重要步驟如下**：

1. **✅ 勾選「Add python.exe to PATH」**（安裝視窗最下方的選項）
   - 這是最重要的步驟，沒勾選會導致無法在終端機執行 Python 指令
   
2. **✅ 勾選「Install pip」**（通常預設已勾選）
   - pip 是 Python 的套件管理工具，用於安裝相依套件

3. 點擊 **「Install Now」** 開始安裝

4. 安裝完成後，點擊 **「Close」** 關閉視窗

#### 3. 驗證 Python 安裝

開啟 PowerShell（Windows 搜尋列輸入 `powershell` 並執行），輸入以下指令確認安裝成功：

```powershell
python --version
```

若顯示版本號（例如 `Python 3.12.0`），代表安裝成功。

若顯示「無法辨識 'python' 詞彙」，代表安裝時未勾選「Add python.exe to PATH」，請重新安裝 Python 並確實勾選該選項。

---

### 步驟二：安裝 OCR Server 相依套件

#### 1. 使用 PowerShell 進入 ocr 資料夾

在 PowerShell 中，使用 `cd` 指令切換到 `ocr` 資料夾。假設你的專案資料夾路徑為 `E:\project\chrome-extension`，則輸入：

```powershell
cd E:\project\chrome-extension\ocr
```

> **提示：** 可以直接在檔案總管開啟 `ocr` 資料夾，然後在資料夾空白處按住 `Shift + 右鍵`，選擇「在此處開啟 PowerShell 視窗」。

確認當前目錄是否正確：

```powershell
pwd
```

應該顯示類似 `E:\project\chrome-extension\ocr` 的路徑。

#### 2. 安裝 Python 相依套件

在 `ocr` 資料夾中執行以下指令，安裝所需的 Python 套件：

```powershell
pip install -r requirements.txt
```

安裝過程會下載並安裝 `ddddocr`、`flask`、`flask-cors` 等套件，需要數十秒至數分鐘（視網路速度而定）。

---

### 步驟三：啟動 OCR Server

在同一個 PowerShell 視窗（仍在 `ocr` 資料夾中），執行以下指令啟動 OCR Server：

```powershell
python ocr_server.py
```

若顯示以下訊息，代表服務已成功啟動：

```
 * Running on http://0.0.0.0:5511
 * Running on http://127.0.0.1:5511
 * Running on http://172.19.0.1:5511
按下 Ctrl+C 停止伺服器
```

**請保持此視窗開啟**，關閉視窗會停止 OCR 服務。

---

### 步驟四：確認服務正常運作

開啟瀏覽器，訪問：

```
http://localhost:5511/health
```

若看到以下回應，代表服務正常：

```json
{"status": "ok", "message": "OCR Server 運行中"}
```

---

### 步驟五：設定 OCR API 網址

在擴充功能的 Tixcraft 基礎設定 → 進階設定 → **OCR API 網址** 中填入：

```
http://localhost:5511/ocr
```

---

### 關閉 OCR Server

在執行視窗按 `Ctrl + C`，或直接關閉視窗。

## 專案結構

```
chrome-extension/
├── tickethelper/           # Chrome 擴充功能主體
│   ├── manifest.json       # 擴充功能宣告
│   ├── background.js       # Service Worker（視窗管理、腳本重注入、OCR 代理）
│   ├── popup.html          # 控制面板 HTML
│   ├── popup.js            # 控制面板邏輯
│   ├── popup.css           # 控制面板樣式
│   ├── kktix/
│   │   └── kktix-content.js        # KKTIX 搶票核心腳本
│   └── tixcraft/
│       ├── tixcraft-content.js     # Tixcraft 搶票核心腳本
│       └── tixcraft-alert-override.js  # Tixcraft alert 攔截器
└── ocr/                    # OCR Server 原始碼
    ├── ocr_server.py       # OCR API 伺服器
    ├── requirements.txt    # Python 依賴套件
    ├── start.bat           # Windows 啟動腳本
    └── start.sh            # Mac/Linux 啟動腳本
```
